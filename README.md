# 🧭 출장 길잡이 (chuljang-giljabi)

출장 동선을 자동으로 정리하고, **네이버지도·카카오맵 무료 보조 확인** 기능을 제공하는 내부 업무용 웹앱입니다.
([paju-giljabi](https://github.com/smile7962/paju-giljabi) 형식을 바탕으로 출발지·복귀지 기능을 일반화한 버전)

## 실행
`index.html` 을 브라우저로 열기만 하면 됩니다. 빌드·서버·API 키가 필요 없습니다.

## 주요 기능
- **출발 지점 자유 입력 + ★ 즐겨찾기 저장**: 아무 주소나 입력해 출발지로 사용, 자주 쓰는 출발지는 저장 목록에서 선택 (이 브라우저 localStorage에 저장)
- **복귀 지점 분리 설정**: 기본은 출발지로 복귀(왕복 최적화), 체크하면 다른 복귀지 지정 가능 → **출발 고정·복귀 고정 편도 최적화**로 계산
- 출장지 주소 입력 (경유 **최대 15곳**) — 자동 접두 지역명(기본 "경기도 파주시") **수정·삭제 가능**, 마지막 사용 출발지·접두어 자동 기억
- **도로 기반 최적 경로 계산** (카카오/OpenStreetMap 좌표변환 + OSRM 도로 경로) — 실거리·주행시간·최적 방문순서
  - 실패 시 자동으로 **임시 계산(입력 순서)** 으로 폴백
- **지도 표시**: 최적 경로를 지도에 그려서 출력 (Leaflet + OpenStreetMap 타일 + OSRM 도로 경로, 무료·무키)
- 총 출장지 · 총 주행 시간 · 총 이동거리 요약
- 순서 카드(번호 + 주소) + `이전→여기 길찾기`(네이버/카카오, **이전=출발·현재=목적지** 자동입력, 스마트폰 앱 연동)
- 출장지 목록 **초기화** 버튼

## 도로 기반 최적 경로 설정 (지오코딩)
도로 기준 최적순서·실거리·실시간 계산은 **키 없이 바로** 동작합니다(OpenStreetMap). 한국 주소 정확도를 높이려면 카카오 JS 키를 선택적으로 넣습니다.

| 단계 | 사용 서비스 | 키 | 비고 |
|---|---|---|---|
| 주소 → 좌표 (기본) | OpenStreetMap (Nominatim) | **불필요** | 브라우저 직접 호출 OK, 정확도 보통, 1초 1요청 정책 준수(자동 지연) |
| 주소 → 좌표 (정확도↑) | 카카오 지도 JS SDK | 카카오 JS키(무료) | 한국 주소 정확. 도메인 등록 필요 |
| 도로 거리·시간 + 최적순서 | OSRM `/trip` 공개 서버 | 불필요 | 자동차 도로 기준 추정 |

### 카카오 JS 키(선택) 사용 시
1. https://developers.kakao.com → 애플리케이션 추가 → **JavaScript 키** 복사
2. 앱 → **플랫폼 → Web → 사이트 도메인**에 접속 주소 등록 (로컬은 `http://localhost`)
3. 앱의 **⚙️ 지오코딩 설정 (고급)**에 JS키 붙여넣기 (키는 **이 브라우저(localStorage)에만** 저장, 서버 전송 없음)
4. **🔌 연결 테스트**로 정상 여부 확인

> 거리·시간은 자동차 도로 기준 추정치이며 실제 교통상황과 다를 수 있습니다. 공개 서버(OSRM/Nominatim)는 소량·보조 용도로만 사용하세요.

## 무료 사용 원칙 (중요)
지도앱 연동은 **무료 공개 URL 방식만** 사용합니다. 유료 API·키·제휴·비공식 호출·웹 스크래핑을 쓰지 않습니다.
지도앱 연동은 **보조 기능**이며, 출장 길잡이의 출장 순서 계산을 대체하지 않습니다.
